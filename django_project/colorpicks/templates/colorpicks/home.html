Demo of integrating traditional and realtime views

The standard Django View
This view uses a combination of generic list and form views, to handle changes in the traditional request/response cycle.
Backbone.js with ajax
In this version, backbone is used to structure the app on the page, and changes are synced back to the server via ajax calls to Django Rest Framework resources.
Backbone.js with websockets and collections
The main demo. This uses the notion of a collection with predicates on the server, connected to socketio through redis-pubsub to provide a fully duplex and realtime web experience. Because the collections are integrated with Django's model signals, they propogate model changes made through other channels.
